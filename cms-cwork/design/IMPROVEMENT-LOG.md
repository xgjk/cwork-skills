# cms-cwork Skill 改进日志

## 2026-04-04 - 待办管理流程优化

### 问题反馈
用户：张成鹏（玄关健康总经理）
时间：2026-04-04 07:35-07:40

### 核心问题
1. ❌ **内容过于繁杂和空洞**：生成的建议/审批内容动辄几百字，缺少针对性
2. ❌ **流程错误**：用户提出修改后，直接修改并发送，没有再次确认
3. ❌ **角色认知不足**：未充分认识到用户意见是最终决策

### 改进方案
1. ✅ **内容精简**：每个建议/审批内容严格控制在 **50字以内**
2. ✅ **流程优化**：
   - 生成内容 → 展示 → 等待反馈
   - 用户提出修改 → 修改内容 → **再次展示** → 等待确认
   - 只有用户明确说"确认"，才执行发送
3. ✅ **角色认知**：用户是玄关健康总经理，意见是最终决策，需慎重对待

### 标准处理流程
```
1. 生成建议/审批内容（≤50字）
2. 展示完整内容
3. 等待用户反馈
4. 用户提出修改？
   ├─ 是 → 修改 → 回到步骤2
   └─ 否 → 等待用户说"确认"
5. 用户说"确认" → 发送
```

## 角色边界明确（2026-04-04 08:06）

### 核心原则
**工厂 Orchestrator 只负责开发，不负责使用**

### 具体规则
1. ✅ **开发职责**：开发、修复、优化、发布 Skill
2. ❌ **禁止使用**：不得直接调用 Skill 发送汇报、查询待办等业务操作
3. 🔄 **用户引导**：如果用户要求使用 Skill，应拒绝并引导至其他 Agent

### 原因
- 避免既是裁判又是运动员
- 确保单一职责原则
- 防止角色混淆导致流程混乱
